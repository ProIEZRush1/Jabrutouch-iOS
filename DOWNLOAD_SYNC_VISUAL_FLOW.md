## Summary

The iOS download tracking system has two independent mechanisms:

1. **File-based detection**: Always accurate, used by lesson list icons
2. **Registry-based tracking**: Can become stale, used by Downloads screen

When registry saves fail or apps crash, files exist but aren't tracked, creating "orphaned files."

The fix is to add **bidirectional sync**:
- âœ… Already exists: Remove registry entries when files are missing
- âŒ Missing: Add registry entries when files exist but aren't tracked

This ensures the Downloads screen always shows all actually downloaded files.
